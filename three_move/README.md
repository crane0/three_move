### 接口文档
>  分析：
1. 页面加载的时候，请求省级单位的所有数据   
当省级单位发生变化，去请求对应的市级单位的所有数据   
当市级单位发生变化，去请求对应的区/县级单位的所有数据   

2. 当请求到数据添加到对应的select标签中
- 请求所有的省级单位的数据
    - 请求方式 ： GET
    - 请求地址 ： '/getProvinces'
    - 请求参数 ： 无
    - 返回的数据格式 ： json
- 请求当前省对应的所有市级单位的数据
    - 当省级select被选中的option发生变化的时候发送请求
    - 请求方式 ： GET
    - 请求地址 ： '/getCities'
    - 请求参数 ： provinceId
    - 返回的数据格式： json
- 请求当前市对应的县/区级的所有数据
    - 当市级select被选中的option发生变化的时候发送请求
    - 请求方式：　GET
    - 请求地址 ： '/getCounties'
    - 请求参数： cityId
    - 返回的数据格式： json


### 优化的问题
1. 要清除上次请求的数据，否则option会叠加
2. 在重新选择省级时，清除市级的同时，还要清除区级
3. 直辖市，有特殊要求。
4. 选择第一个（“请选择xxx”）时，不应该发送请求。
